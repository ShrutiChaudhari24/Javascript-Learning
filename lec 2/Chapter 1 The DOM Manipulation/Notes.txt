Explanation
🔹 1️⃣ What is DOM?
DOM (Document Object Model) is a programming interface that represents an HTML or XML document as a tree structure of objects, allowing JavaScript to access and modify the content, structure, and style of a webpage dynamically.
👉 When a browser loads an HTML page, it converts the HTML into a DOM tree.

🔹 2️⃣ How DOM Works?
Browser reads HTML.
It parses the HTML.
It creates a tree-like structure in memory.
Each HTML element becomes a node (object).
JavaScript can access these nodes using the document object.

Example: 
<body>
  <div>
    <h1>Hello</h1>
  </div>
</body>

DOM Tree:
Document
 └── html
      └── body
           └── div
                └── h1

🔹 3️⃣ Types of Nodes in DOM
Element Node → <div>, <p>, <h1>
Text Node → Text inside elements
Attribute Node → id, class
Document Node → Root of the DOM

🔹 4️⃣ What is DOM Manipulation?
DOM Manipulation means changing the structure, content, or style of a webpage using JavaScript.
Example:
document.getElementById("title").innerText = "Updated";

🔹 5️⃣ Common DOM Methods (Important for Interviews)
getElementById()
getElementsByClassName()
querySelector()
querySelectorAll()
createElement()
appendChild()
remove()

🔹 6️⃣ Why DOM is Important?
Makes webpages dynamic
Enables user interaction
Allows real-time content updates
Forms the foundation of frameworks like React, Angular, and Vue.js

DOM stands for Document Object Model. 
When a browser loads an HTML document, it converts it into a tree-like structure of objects called the DOM. 
Each HTML element becomes a node, and JavaScript can access and manipulate these nodes to dynamically change the content, structure, or styling of a webpage.

✅ Difference Between console.log() and console.dir()
console.log()	                                       |      console.dir()
Prints a readable representation of the object	     |     Prints the object as a JavaScript object (expandable properties)
Best for normal values (string, number, array)	     |     Best for inspecting DOM elements and objects
Shows HTML structure when logging DOM	               |    Shows properties of DOM element as object

🔎 Example With DOM
<h1 id="demo">Hello</h1>

let el = document.getElementById("demo");

console.log(el);
console.dir(el);

🟢 console.log(el)
Shows:
<h1 id="demo">Hello</h1>
👉 It prints the element like HTML.

🔵 console.dir(el)
Shows:
h1#demo
id: "demo"
innerText: "Hello"
innerHTML: "Hello"
style: CSSStyleDeclaration
...

👉 It shows the element as a JavaScript object
👉 You can expand and see all properties

💡 Simple Understanding
console.log() → shows what it looks like in HTML
console.dir() → shows what it is as a JS object

🧠 When To Use What?
✅ Use console.log()
Debug normal values
Print strings, numbers, arrays
See HTML structure

✅ Use console.dir()
Inspect DOM elements deeply
Check properties like:
innerText
innerHTML
style
attributes
event handlers

(1) Selecting elements from html :- this are used to select the elements before changing 
1️⃣ getElementById()
🔹 What it does?
👉 Selects one element by its id.

🔹 Important Rules
ID must be unique
Returns single element
If not found → returns null

🔹 Example
html
<h1 id="title">Hello</h1>

javascript
let el = document.getElementById("title");
console.log(el);

getElementById() selects a single element using its unique id.

2️⃣ getElementsByClassName()
🔹 What it does?
👉 Selects all elements with same class name.

🔹 Important Rules
Many elements can have same class
Returns HTMLCollection
It is a live collection
If not found → returns empty collection

🔹 Example
html
<p class="box">One</p>
<p class="box">Two</p>
<p class="box">Three</p>

javascript
let elements = document.getElementsByClassName("box");
console.log(elements);


⚠ Important Difference
getElementById	          |    getElementsByClassName
Returns one element	      |    Returns multiple elements
Returns object	          |    Returns HTMLCollection
ID must be unique	        |    Class can repeat
Not live concept	        |    Live collection

🔥 What is Live Collection?
If new element is added with same class,
getElementsByClassName() will automatically update.

Example:
let boxes = document.getElementsByClassName("box");
document.body.innerHTML += '<p class="box">New</p>';
console.log(boxes.length); // Automatically increased
👉 That is called Live Collection.

🎯 Short 
getElementById() selects a single element using its id, while 
getElementsByClassName() selects multiple elements using class name and returns a live HTMLCollection.


(3)querySelector()

Selects the first matching element in the document
Uses CSS selectors
Returns a single Element (or null if not found)

Syntax
document.querySelector("selector");
Examples
document.querySelector("p");        // first <p>
document.querySelector(".box");     // first element with class="box"
document.querySelector("#title");   // element with id="title"

Key Points
Stops after the first match
Very common for grabbing one element
If nothing matches → returns null


(4)querySelectorAll()
Selects all matching elements
Uses CSS selectors
Returns a NodeList (not an array)

Syntax
document.querySelectorAll("selector");
Examples
document.querySelectorAll("p");        // all <p> elements
document.querySelectorAll(".box");     // all elements with class="box"

Looping through results
document.querySelectorAll("p").forEach(p => {
  p.style.color = "blue";
});

Key Points
Gets every match
NodeList supports forEach
If nothing matches → returns an empty NodeList


Quick Comparison
| Feature      | `querySelector()` | `querySelectorAll()` |
| ------------ | ----------------- | -------------------- |
| Returns      | One element       | NodeList             |
| Matches      | First only        | All                  |
| Loop needed  | ❌ No              | ✅ Yes                |
| If not found | `null`            | Empty NodeList       |
